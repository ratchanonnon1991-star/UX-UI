# UI Rules

- Use `design-system.md` as the source of truth.
- Do not invent new colors.
- Use Tailwind CSS only for HTML prototypes.
- Do not use backend, database, auth, or real payment integration in prototypes.
- Use mobile-first responsive layout.
- Use consistent spacing from the 4px scale.
- All cards must use the defined radius and border.
- Main CTA must use the primary gradient.
- Use glow only for CTA, selected cards, promotions, and success states.
- Use placeholder images or original mascot/illustration only.
- Do not use copyrighted game characters.
- Forms must include labels, helper text, focus state, and error state.
- Every page must include clear next action.
- Keep checkout flow simple: select game → enter ID → select package → pay → track.
- Output only requested files or code. Avoid long explanations.
